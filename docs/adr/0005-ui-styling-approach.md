# ADR-0005: UIスタイリングアプローチ

## ステータス

採用

## 背景

UIのスタイリング方法を決定する必要があった。開発速度、保守性、パフォーマンス、学習コストを考慮。

## 決定事項

**CSSフレームワークを使用せず、シンプルなカスタムCSSで実装する。**

### 実装方針
- 単一のCSSファイル (`public/style.css`)
- クラスベースのスタイリング
- レスポンシブデザイン（メディアクエリ使用）
- CSSフレームワーク（Bootstrap, Tailwind CSS等）は不使用

## 検討した選択肢

1. **シンプルなHTML/CSS** ← 採用
2. TailwindCSS
3. Bootstrap等の既存CSSフレームワーク

## 理由

### 採用理由
1. **シンプルさ**: UIが単純（一覧ページと投稿フォームのみ）
2. **軽量**: 外部依存なし、ビルドプロセス不要
3. **保守性**: カスタムCSSは理解しやすく変更が容易
4. **パフォーマンス**: 余分なCSSがない、ファイルサイズが小さい
5. **学習コスト**: 標準的なCSSのみで完結

### TailwindCSSを不採用にした理由
- セットアップが複雑（PostCSS、ビルドプロセス）
- UIが単純なためユーティリティクラスの恩恵が少ない
- プロジェクトの複雑性を増加させる

### Bootstrapを不採用にした理由
- オーバーキル（必要な機能に対して過剰）
- カスタマイズが煩雑
- ファイルサイズが大きい

## スタイリングの特徴

### デザイン原則
- **ミニマル**: 装飾を最小限に
- **読みやすさ**: 適切な余白とコントラスト
- **レスポンシブ**: モバイルファーストではないが、モバイル対応

### 主要コンポーネント
```css
.btn           # 汎用ボタン
.btn-primary   # プライマリボタン
.message-card  # メッセージカード
.post-form     # 投稿フォーム
```

### カラースキーム
- プライマリカラー: `#3498db` (青)
- テキスト: `#2c3e50` (ダークグレー)
- 背景: `#f5f5f5` (ライトグレー)
- ヘッダー: `#2c3e50`

## 結果

### 良い点
- ビルドプロセス不要
- CSSファイルが小さい（約200行）
- 変更が容易
- 依存関係なし

### 悪い点
- 大規模なUIには向かない
- コンポーネントの再利用性が低い
- デザインシステムの恩恵がない

### トレードオフ
- 開発速度とシンプルさを優先
- スケーラビリティは犠牲に（小規模アプリには問題なし）

## 今後の拡張

UIが複雑化した場合の選択肢：
1. CSS Modulesの導入
2. TailwindCSSへの移行
3. コンポーネントライブラリ（Reactと組み合わせて）

現時点では、このシンプルなアプローチで十分。

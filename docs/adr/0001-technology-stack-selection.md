# ADR-0001: 技術スタックの選定

## ステータス

採用

## 背景

シンプルなメッセージボードアプリケーションを構築するにあたり、開発の容易性、保守性、パフォーマンスを考慮した技術スタックの選定が必要だった。

## 決定事項

以下の技術スタックを採用する：

- **Webフレームワーク**: Hono.js + @hono/node-server
- **データベース**: SQLite3 (better-sqlite3)
- **言語**: TypeScript
- **開発ツール**: tsx
- **テンプレート**: hono/html
- **テスト**: Node.js Test Runner

### 選定理由

#### Hono.js
- 軽量で高速なWebフレームワーク
- Express風のシンプルなAPI
- TypeScriptのサポートが優れている
- Node.js環境で動作可能（@hono/node-server経由）

#### SQLite3
- セットアップが不要（ファイルベース）
- 小規模アプリケーションに最適
- 開発環境での取り扱いが容易
- better-sqlite3による同期APIで扱いやすい

#### TypeScript
- 型安全性による開発効率の向上
- IDEサポートの充実
- リファクタリングの容易性

#### Node.js Test Runner
- 追加のテストフレームワーク不要
- Node.js v18以降に標準搭載
- シンプルなテストに十分な機能

## 結果

### 良い点
- 依存関係が少なくシンプル
- セットアップが容易
- 開発サーバーの起動が高速
- TypeScriptによる型安全性

### 悪い点
- SQLiteはスケーラビリティに限界がある（本番環境では別のDBへの移行が必要になる可能性）
- Honoはまだ比較的新しいフレームワーク（エコシステムがExpressより小さい）

### トレードオフ
- 開発の容易性を優先し、大規模スケールは後回し
- シンプルさを重視し、機能豊富なフレームワークは避けた
